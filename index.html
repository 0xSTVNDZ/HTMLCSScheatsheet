HTML/CSS Cheatsheet

HTML

Remember to google the available attributes for tags.

Sectioning

Tag Name Description
<html> html Represents the root of an HTML document. Referred to as root element.
<body> body Represents the content of an HTML document.
<article> article Represents a self contatined composition in a document which is intended to be indenpendent.
<address> address Provides contact information for a person or people.
<aside> aside Portion of a document whose content is only indirectly related to the document's main content.
<footer> footer Typically contains info about author, copyright, or links to related documents.
<div> div Generic container for flow content.
<header> header Represents introductory content. Navigational aids.
<main> main Represents the dominant content of the body of a document.
<nav> nav Represents a section of a page whose purpose is to provide navigation links.
<section> section Represents generic standalone sections of a document.
<h1-h6> h1-h6 Represents six levels of section headings. h1 is highest and h6 is the lowest.

Metadata

Tag Name Description
<base> base Specifies the base URL to use for all relative URLs in a doument
<head> head Contains machine readable information about the document such as title, scripts, and style sheets.
<link> link Specifies relationships between current document and an external resource.
<meta> meta Represents data that cannot be represented by other HTML elements such as base, link, style, title.
<style> style Contains style information for a document or part of a document.
<title> title Defines the documents title that is shown in a browswers title bar or page's tab.

Text Content

Tag Name Description
<blockquote> blockquote Indicates that the enclosed text is an extended quotation.
<dl> dl The element encloses a list of groups of terms (specified using the dt element) and descriptions (provided by dd elements).
<dt> dt Specifies a term in a description or definition list, and as such must be used inside a dl element.
<dd> dd provides the description, definition, or value for the preceding term (dt) in a description list (dl).
<figcaption> figcaption Represents a caption or legend describing the rest of the contents of its parent figure element.
<figure> figure Represents self-contained content, potentially with an optional caption, which is specified using the figcaption element.
<hr> hr Represents a thematic break between paragraph-level elements: (horizonal line) for example, a change of scene in a story, or a shift of topic within a section.
<p> p Represents a paragraph.
<ol> ol Represents an ordered list of items.
<ul> ul Represents an unordered list of items.
<li> li Used to represent an item in a list.

Inline Semantics

Tag Name Description
<a> a Creates a hyperlink to web pages, files, email addresses, locations in the same page, or anything else a URL can address.
<br> br Produces a line break in text.
<cite> cite Used to describe a reference to a cited creative work, and must include the title of that work.
<em> em Marks text that has stress emphasis.
<i> i These have been presented using italicized type.
<strong> strong Indicates that its contents have strong importance, seriousness, or urgency. Typically render the contents in bold type.
<sub> sub Specifies inline text which should be displayed as subscript for solely typographical reasons.
<sup> sup Specifies inline text which is to be displayed as superscript for solely typographical reasons.
<u> u This is rendered by default as a simple solid underline, but may be altered using CSS.
<code> code displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code.

Images and Multimedia

Tag Name Description
<audio> audio Used to embed sound content in documents. It may contain one or more audio sources, the browser will choose the most suitable one.
<img> img Embeds an image into the document.
<video> video Embeds a media player which supports video playback into the document. You can use video for audio content as well, but the audio element may provide a more appropriate user experience.
<embed> embed Embeds external content at the specified point in the document.

Tables

Tag Name Description
<caption> caption Specifies the caption (or title) of a table.
<table> table Represents tabular data
<thead> thead Defines a set of rows defining the head of the columns of the table.
<th> th Defines a cell as header of a group of table cells.
<tbody> tbody Encapsulates a set of table rows (tr elements), indicating that they comprise the body of the table (table).
<tr> tr Defines a row of cells in a table.
<td> td Defines a cell of a table that contains data.
<tfoot> tfoot Defines a set of rows summarizing the columns of the table.

Forms

Tag Name Description
<button> button Represents a clickable button, used to submit forms or anywhere in a document for accessible, standard button functionality.
<datalist> datalist Contains a set of option elements that represent the permissible or recommended options available to choose from within other controls.
<fieldset> fieldset Used to group several controls as well as labels (label) within a web form.
<form> form Represents a document section containing interactive controls for submitting information.
<label> label Represents a caption for an item in a user interface.
<input> input Used to create interactive controls for web-based forms in order to accept data from the user.
<legend> legend Represents a caption for the content of its parent fieldset.
<meter> meter Represents either a scalar value within a known range or a fractional value.
<progress> progress Displays an indicator showing the completion progress of a task, typically displayed as a progress bar.
<select> select Represents a control that provides a menu of options.
<textarea> textarea Represents a multi-line plain-text editing control.
<option> option Used to define an item contained in a select, an optgroup, or a datalist element.

Misc

Tag Name Description
<script> script Used to embed executable code or data; this is typically used to embed or refer to JavaScript code.
<del> del Represents a range of text that has been deleted from a document.
<ins> ins Represents a range of text that has been added to a document.
